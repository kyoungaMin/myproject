# Claude 개발 가이드

이 문서는 Claude Code가 프로젝트 작업 시 따라야 할 규칙과 가이드라인을 정의합니다.

---

## 📝 개발일지 작성 규칙

### 기본 원칙
**모든 작업 시 자동으로** 개발일지를 "개발일지" 폴더에 날짜별 MD 파일로 작성합니다.

- 작업 시작 시: 해당 날짜의 개발일지 파일 자동 생성 또는 열기
- 작업 진행 중: 실시간으로 개발일지 업데이트
- 각 작업 단계마다 진행 상황 기록
- 사용자가 별도로 요청하지 않아도 자동으로 작성

### 파일 형식
- **위치**: `개발일지/YYYY-MM-DD.md`
- **명명 규칙**: 날짜 형식 (예: `2024-12-24.md`)

### 필수 포함 내용

#### 1. 작성 시각
- 작업 시작 시간 기록
- 주요 작업 단위별 시간 기록

#### 2. 해결하고자 한 문제
- 현재 작업의 목표와 해결하려는 문제 명확히 기술
- 각 문제에 대한 접근 방법 기록

#### 3. 해결된 것
- ✅ 체크마크로 표시
- 완료된 작업 상세 내역
- 구현된 기능 및 수정 사항

#### 4. 해결되지 않은 것
- ⏳ 표시로 미해결 상태 표시
- 블로커 및 이슈 사항
- 추가 조사가 필요한 항목

#### 5. 향후 개발을 위한 컨텍스트
- 프로젝트 현황 스냅샷
- 다음 단계 계획
- 주요 의사결정 사항 및 근거
- 참고할 만한 정보

## 🧪 백엔드 개발 및 테스트 규칙

### 테스트 우선 원칙
백엔드 로직에서 API 함수나 클래스 등을 구현할 때마다 **반드시** 테스트를 철저히 수행합니다.

### 개발 프로세스

1. **구현**
   - API 함수 또는 클래스 작성
   - 에러 핸들링 포함

2. **테스트 작성 및 실행**
   - 단위 테스트 작성
   - 통합 테스트 (필요시)
   - 엣지 케이스 검증
   - 에러 케이스 테스트

3. **검증**
   - 모든 테스트 통과 확인
   - 코드 리뷰 (필요시)
   - 성능 검증 (필요시)

4. **다음 단계 진행**
   - **검증이 완료된 경우에만** 다음 단계로 진행
   - 미검증 코드는 절대 다음 단계로 넘기지 않음

### 테스트 체크리스트
- [ ] 정상 동작 케이스
- [ ] 경계값 테스트
- [ ] 에러 핸들링
- [ ] 입력 검증
- [ ] 반환값 검증
- [ ] 부작용(side effects) 확인

## 📌 작업 진행 시 주의사항

1. **단계별 검증**: 각 단계가 완료되면 반드시 검증 후 다음 단계로 진행
2. **문서화**: 모든 중요한 결정사항과 작업 내역을 개발일지에 기록
3. **품질 유지**: 테스트되지 않은 코드는 미완성 코드로 간주
4. **컨텍스트 보존**: 향후 개발을 위해 충분한 정보를 남김
